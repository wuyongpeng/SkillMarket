How to Build a Basic Harness

  1. Define output schema — use Pydantic / Zod to enforce structured responses from the model.
  2. Add an evaluator — write a function that scores each output (correctness, format, safety).
  3. Instrument the loop — log every call: prompt, output, latency, token cost.
  4. Add retry + fallback — on eval failure, retry with a modified prompt or route to a fallback model.
  5. Close the loop — feed failures into a dataset for fine-tuning or few-shot examples.
