Common Harness Failures

No output validation
Accepting raw LLM text without schema enforcement leads to silent downstream failures. Always parse and validate before passing output to the next step.
Missing observability
Without per-call logging you cannot diagnose regressions after model updates. Log prompt hash, model version, latency, and token count at minimum.
Infinite retry loops
Retrying on every failure without a max-attempts cap or exponential backoff burns tokens and blocks pipelines. Set hard limits.
Evaluation drift
Evaluators that are not versioned alongside prompts produce inconsistent scores over time. Treat evaluators as first-class code artifacts.
